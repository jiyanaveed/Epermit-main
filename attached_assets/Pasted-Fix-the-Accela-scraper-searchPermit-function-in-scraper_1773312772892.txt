The problem is in step 4 where it scans elements. The permit 
number link in the Accela grid might have extra whitespace, 
newlines, or be inside nested elements. The current code does:
  const text = el.textContent.trim();

This fails because:
1. textContent on a parent element might include other text 
   around the permit number
2. The permit number link is inside a table cell with class 
   links — textContent grabs everything
3. The link text might have \n or \t characters

Replace the entire step 4 scan with this approach:
